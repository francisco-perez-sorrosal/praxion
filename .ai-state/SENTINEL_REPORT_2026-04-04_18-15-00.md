### Code Health

CH01 (LLM): Sampled 5 hook implementation files (`memory_gate.py`, `validate_memory.py`, `detect_duplication.py`, `capture_memory.py`, `promote_learnings.py`).

**Finding**: `memory_gate.py` and `validate_memory.py` share a structurally duplicated `_scan_transcript()` function. Both:
1. Open a transcript file path
2. Iterate through JSON lines
3. Parse each line as a turn dict
4. Filter for `type == "assistant"` turns
5. Walk `content` blocks for `type == "tool_use"`
6. Count tools matching `SIGNIFICANT_TOOLS` and `REMEMBER_TOOL_SUBSTRING`
7. Return a tuple of counts

The `validate_memory.py` version adds `wrote_learnings` tracking and checks `tool_input.file_path` for "LEARNINGS", but the core scanning logic is ~85% identical. Both also share identical constants (`MIN_EDITS_FOR_SIGNIFICANT = 3`, `REMEMBER_TOOL_SUBSTRING`, `SIGNIFICANT_TOOLS`), the same retry/pass-through pattern, and nearly identical error message templates.

Other sampled files (`detect_duplication.py`, `capture_memory.py`, `promote_learnings.py`) are well-differentiated with distinct responsibilities and no significant duplication.
