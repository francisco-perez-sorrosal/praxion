## Future Paths

Directional possibilities for where the project could go. Paths may be compatible or mutually exclusive -- this section maps the option space, not a single roadmap.

- **Multi-assistant expansion** -- Extend beyond Claude Code to Cursor, ChatGPT, and other assistants. Would require abstracting assistant-specific config into adapter layers while keeping shared assets (skills, commands, agents) portable.
- **Community plugin ecosystem** -- Evolve from personal config repo to a distributable plugin marketplace with versioning, dependency management, and discovery.
- **Project template generation** -- Use the accumulated skills and rules as a foundation for scaffolding new projects with pre-configured AI assistant setups.
- **Pipeline quality loop** -- Close the pipeline with a verification stage, enabling the full cycle from ideation through verified result. The verifier agent and sentinel with persistent reports are in place. The sentinel->promethean dependency creates a feedback loop: health findings inform ideation. Future iterations could add security review, performance analysis, and cross-feature retrospective capabilities.
- **Ecosystem observability** -- Extend Task Chronograph beyond pipeline activity tracking into quality monitoring. The sentinel's persistent report log (`SENTINEL_LOG.md`) and timestamped reports provide historical data. Could evolve into a full quality dashboard with trend analysis, degradation alerts, and per-artifact health sparklines.
- **Language skill expansion** -- Add language-specific development skills beyond Python, starting with TypeScript (Claude Code plugin/MCP server ecosystem) and eventually Rust. Each follows the `<language>-development` naming pattern established by the python rename.
- **Multi-project memory** -- Split memory into global (`~/.claude/memory.json`) and project-local (`.ai-state/memory.json`) tiers, mirroring Claude Code's own CLAUDE.md hierarchy. User/tool entries route to global; project/learnings stay local. Identified during memory v2.0 research but deferred as a separate architectural decision.
- **External knowledge integration** (new, 2026-03-30) -- Bridge Praxion's closed-loop pipeline with curated community knowledge sources. The `external-api-docs` skill (context-hub integration) is the first step. Future extensions: `setup-chub` command for guided installation (Phase 2), researcher agent with explicit chub awareness (Phase 2), registry publishing for ecosystem-independent skills (Phase 3), and eventually Rust reimplementation evaluation for team features (Phase 3). Architectural patterns worth adopting regardless: annotation persistence, multi-source registry merge, BM25 search indexing, content trust tiers.
- **Developer onboarding as cross-project capability** (new, 2026-04-06) -- The Project Cartographer's methodology could extend beyond single-project onboarding to multi-project understanding (e.g., understanding how microservices relate to each other), organizational-level architectural patterns, and team knowledge transfer. The learning map could become a shared artifact that teams curate together.
