## Model Selection & Capabilities

### Claude Opus 4.6: Practical Guide
[📖 Read](https://www.the-ai-corner.com/p/claude-opus-4-6-practical-guide)

**When to read:** Deciding between Claude models, understanding adaptive reasoning, or planning multi-artifact work.

**Key takeaways:**
- Opus 4.6 excels at ambiguous, large-scale, multi-artifact tasks with compound reasoning
- Four capability shifts: adaptive effort management, long-context performance (1M tokens), multi-step reasoning, consistent writing quality
- Best for strategic problems, architecture decisions, financial analysis, complex research, and long-form co-authoring
- Recommended over faster models when quality and reasoning depth matter

**Praxion connection:** Opus 4.6 is the recommended model for agent-driven pipelines (systems-architect, verifier, implementation-planner). Reserve Haiku for lightweight tasks; Sonnet for standard workflows.

## Cowork Setup & Configuration

### Claude Cowork Setup: The Complete Guide
[📖 Read](https://www.the-ai-corner.com/p/claude-cowork-setup-guide)

**When to read:** Configuring Claude Desktop for persistent context, setting up project-specific instructions, or building role-based workflows.

**Key takeaways:**
- Five setup layers: context files (identity/preferences), global instructions (session defaults), folder instructions (project-scoped context), plugins (role packs), connectors (live integrations)
- Critical gap: Cowork has no memory between sessions — proper setup bridges this through persistent context files
- Templates provided for about-me, brand-voice, and working-style documents
- 10 ready-to-use workflows: weekly planning, meeting prep, document drafting, research synthesis, content pipelines, data analysis, client deliverables, email drafting, reports, status updates

**Praxion connection:** Praxion's memory MCP and CLAUDE.md files solve the session-memory gap. This guide complements Praxion's persistent knowledge approach.

## Philosophy & Market Context
